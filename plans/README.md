## Findings considered and rejected / deferred

Recorded so the next audit doesn't re-litigate them.

- **CRDT write authorization (any client can forge other users' availability)**
  — inherent to the local-first Yjs design; server-side update filtering would
  require per-connection identity and deep y-partyserver surgery, out of
  proportion for a friend-group scheduler where the event URL is the trust
  boundary. Revisit only if abuse actually appears.
- **`.partykit/` committed to git** — false finding; the directory is local
  state, not tracked (verified with `git ls-files`).
- **react-scan ships to production bundle** — false; the import is statically
  dead (`SCAN = false` + NODE_ENV check) and Vite drops it. The *dependency
  placement* is real and handled in plan 006.
- **Playwright exact-pin overrides** — by design (commit af466f6 dedupes
  playwright versions); not a defect.
- **Coverage collected on Chromium only** — by design; V8 coverage API is
  Chromium-only. Firefox still runs the specs as a regression gate.
- **CORS dev-only block keyed on `process.env.NODE_ENV`**
  (`party/editor.partyserver.ts:106`) — accepted risk: build-time define in the
  Worker bundle; wildcard would require both a misbuild and a useful
  cross-origin target. Not worth a plan; noted for reviewers.
- **Vite 6→8 / ESLint 9→10 / plugin-react 4→6 / React 19.0→19.2 / nyc 18 /
  tailwind 4.3 major+minor migrations** — deferred as a batch; plan 006 patches
  security within current majors first. Do majors one-per-PR with full e2e.
- **EventDetails.tsx god-component split (772 lines)** — real debt, but L
  effort + HIGH regression risk on the core feature; do it *after* 001/003 land
  (tests first), as its own initiative with the memoization work below.
- **Unmemoized derived state / per-cell re-renders in the availability grid** —
  real but bounded (grids are ~30 days × handful of users); fold into the
  EventDetails refactor rather than sprinkling `useMemo` into a file about to
  be restructured.
- **`min={5}` on the history slider + unguarded `updates[index]`**
  (`app/ui/EventHistory/index.tsx:208,229`) — real papercut; too small for a
  plan. Fix opportunistically in any PR touching that file (one-line:
  `min={0}`, guard `updates[index]?.clock`).
- **Two component libraries (radix-ui + @base-ui-components)** — real
  duplication, one component each; consolidation is cosmetic until either
  library blocks an upgrade. Deferred.
- **occupancy count race / fire-and-forget updates** — informational dashboard
  only; self-corrects on next connection event. Not worth the CAS machinery.
- **partysocket/partyserver 0.x abandonment risk** — watch item, no action;
  both are Cloudflare-maintained.

## Direction options (maintainer's call, not ranked against bugs)

Grounded suggestions from the audit, for when the bug/security queue clears:

1. **Event expiry/deletion as a product feature** — plan 007 adds silent TTL;
   the visible half (creator delete button, "this event expired" tombstone) is
   a natural follow-up the storage model now supports.
2. **Per-event cursor-party rooms** — `todo.gitignored.md` already calls for
   it; currently all events share one global cursor room
   (`app/ui/CursorPartyScript.tsx:14`). Hours of work, visible polish.
3. **Timezone display support** — plan 003 made the math UTC-consistent
   (react-day-picker turned out to already be UTC-pinned, so no boundary
   helpers were needed); showing the event in a chosen IANA zone (crab.fit
   parity) is the natural next layer on that invariant.
4. **.ics export** — todo.gitignored.md "Big Features" lists calendar saving;
   right-click → download .ics for the winning date is a contained, offline
   feature.
5. **Hour-level granularity** — the README explicitly concedes this to
   crab.fit; largest product lift, schema + grid redesign; spike before
   committing.
